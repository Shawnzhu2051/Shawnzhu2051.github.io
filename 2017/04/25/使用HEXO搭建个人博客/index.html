使用HEXO搭建个人博客

2017-04-25 Web开发

说在前面：wordpress真的是大坑，掉进去了都走不出来的大坑，安装花了两天，前前后后配置也花了好几天，最后还是有几个影响极大却迟迟修复不了的bug，遂弃疗。

正好这周考完了半期考试，不是特别忙，总算找到一个时间来好好管理一下个人博客了。这一次我选择了一个可能是最简单的博客框架–HEXO，来重新搭建博客。

HEXO是基于Github的一个 fast, simple & powerful blog framework (官网原话)，它最大的好处就是不需要你自己去准备服务器（运营个人博客最大的花销就是服务器了），甚至域名也不需要，只需要你有一个Github账号，以及一台能ssh连上Github的电脑就行了。

PS：以下所有的本地操作适用于MacOS系统，Linux系统操作基本相同，Windows系统在命令行的一些操作与另外两种系统不同，请自行百度。

Github上的准备工作

如果没有Github账号的话先注册，Github自己注册就行。

然后新建仓库(New repository)，仓库的名称输入：

Your_Github_Name.github.io

下一步是添加密钥，这个ssh密钥的作用是通过ssh远程连接，使你可以从命令行里把文件push进你的GitHub仓库，从而实现博客的配置，更新和修改。

需要注意的一点是，Github密钥一台电脑最好只连接一个，如果相同一台电脑通过ssh密钥连接多个Github账号的话，非常麻烦，不要问我为什么知道这个：）

在命令行中输入

ssh-keygen -t rsa -C "你的Github注册邮箱地址"

一路enter就行，生成的密钥存放在~/.ssh里，这是一个隐藏的文件夹，可以在命令行中使用

ls -a

来显示隐藏的文件夹。

打开.ssh文件夹中的id-rsa.pub文件，用记事本、vi、gedit、sublime、atom什么都可以，然后复制该.pub文件里的内容，进入https://github.com/settings/ssh，然后新建ssh key，Title可以随便填，把你复制的内容粘贴到Key，然后add SSH key即可。

本地的准备工作

首先是安装node.js，可以通过命令行安装（pip 或 apt-get），但是可能版本有些问题，建议直接进去node.js官网下载源码安装。推荐使用LTS版本。

然后需要安装git。如果你不确定你的电脑里有没有git可以在命令行输入

git --version

来查看是否有安装。如果没有的话，也是可以通过命令行安装，或者去git官网下载源码安装。

HEXO安装与配置

在hexo的安装配置中可能有一些操作需要用到root，我没有去仔细深究，所以遇到报错可以试试加上sudo看看行不行，也可以直接在root下安装。

先在本地新建一个文件夹，用来安装放置hexo。

mkdir 文件夹名
cd 文件夹名

然后下载安装hexo

npm install -g hexo-cli

速度可能会比较慢，耐心等候，掉线了就重新来一次。

安装好之后，我们需要初始化hexo

hexo init 文件夹名
cd 文件夹名
npm install

完成后，在文件夹里找到_config.yml文件，这是我们博客的站点配置文件，里面存放着博客的一些基本信息，打开并修改如下参数：

# Site
title: Your_Blog_Name 
subtitle: Your_Blog_Subname
description: #可填可不填
author: Your_Name
language: zh-CN #在某些主题里可能会要求你填写zh-Hans
timezone: Asia/Shanghai

然后拉到最下面：

# Deployment
## Docs: https://hexo.io/docs/deployment.html
deploy:
      type: git
      repo: https://github.com/你的github仓库地址.git 
      # 在Github上你新建的仓库处可以找到
      # 比如https://github.com/shawnzhu2051/shawnzhu2051.github.io.git
      branch: master

需要注意的是，yml文件对格式要求很严格，注意在冒号后都是有且仅有一个空格的。

至此hexo的基本配置就搭建好了。

HEXO的基本操作

  • 新建文章：

    hexo new '文章题目'

    在本地文件夹的/source/_post里可以找到一个新建的markdown文件，只需要在里面写，保存即可。

  • 新建页面：

    hexo new page '页面名'

    然后同样可以找到一个同名的文件夹，里面有一个index.md文件。
    需要注意的是，hexo默认是只有Home 和 Achieve两个页面的，所以诸如分类(categories)，标签(tags)，关于（about）等页面都是没有的，请在上述的index.md文件中，title后面添加上

    layout: "categories"/"tags"/"about"

  • 本地预览

    hexo server

    然后再你自己的浏览器中输入

    localhost:4000

    或者

    127.0.0.1:4000

    就能看到你的博客了。

  • 博客发布

    先执行一次命令，解决掉deploy的bug

    npm install hexo-deployer-git --save

    之后每次发布就只用执行：

    hexo generate
hexo deploy

    这个时候就可以在网络上看到你的博客了，域名默认是

    Your_Github_Name.github.io

    也就是你之前创建的那个仓库的名字

    如果想要关联到自己购买的域名的话也可以，具体的请百度。
